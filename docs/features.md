## 核心功能

### 1. 自动移除

| 功能 | 说明 |
| --- | --- |
| 自动点移除 | 针对当前可见第一项粉丝，自动点击“移除”按钮 |
| 自动点确认 | 等待确认层可见后，自动点击“确认移除” |
| 同时处理 | 每轮默认同时处理最多 `5` 个当前可见的非“相互关注”目标 |
| 自动触底 | 当前已加载出来的项目里，排除“相互关注”后，非互关目标少于 `15` 个时自动触发一次触底 |
| 底部收尾 | 检测到“暂时没有更多了”后停止继续下滑，继续处理当前可见区域里的可删目标 |
| 昵称兼容 | 表情昵称和空白昵称粉丝仍会被识别为当前行的可删目标 |
| 每日上限 | 按当前平台限制，每天上限移除 `2500` 人 |
| 循环处理 | 删除一项后继续处理下一项，直到达到本轮上限、暂停或当前窗口没有可处理项 |

### 2. 悬浮控制面板

| 功能 | 说明 |
| --- | --- |
| 开始 | 启动自动处理循环 |
| 暂停 | 立即停止后续操作 |
| 状态统计 | 面板内显示当前状态、已移除数量、已跳过数量 |
| 固定策略 | 默认执行间隔为 `0`，每轮默认同时处理最多 `5` 人 |

### 3. 阈值触底定位

| 功能 | 说明 |
| --- | --- |
| 阈值判断 | 当前已加载出来的项目里，排除“相互关注”后的目标数量少于 `15` 个时触发 |
| 单次触底 | 每次只执行一次滚动到底，再等待列表稳定后继续处理 |
| 可见区处理 | 未达到阈值时，脚本仍然只处理当前可见区域内的可删目标 |
| 末尾处理 | 页面出现“暂时没有更多了”后，处理范围仍然保持在当前可见区 |
| 昵称异常处理 | 即使昵称渲染为表情或空白，仍按粉丝项本身识别，不会误并到其他行 |

### 4. 自动跳过相互关注

| 功能 | 说明 |
| --- | --- |
| 自动识别 | 通过列表项中的按钮文本识别“相互关注” |
| 直接跳过 | 当前可见区域内遇到“相互关注”时不会执行移除 |
| 无需白名单 | 不再依赖手工录入白名单 |

### 5. 页面范围限制

| 功能 | 说明 |
| --- | --- |
| 精确生效 | 仅匹配 `https://www.douyin.com/user/self*` |
| 降低误触 | 不会在推荐页、搜索页、他人主页等场景注入 |

## 适用场景

- 需要批量清理历史粉丝
- 需要保留“相互关注”用户，不希望手工逐个判断
- 需要在可删目标变少时自动补一次触底加载
- 需要用可调间隔慢速调试或快速连续执行
- 需要注意平台每天最多只能移除 `2500` 人

## 当前实现说明

- 选择器优先使用抖音粉丝弹窗内的 `data-e2e` 标识与按钮文案
- 为降低 class 名变动影响，脚本对“移除”“确认移除”“相互关注”采用文本识别
- 确认按钮采用显隐检测，避免在确认层未展开时误点
